# Muse-image

> 发掘用户想象的对话式文生图工具 —— AI 先出方案，你做主编。

## 这是什么

**Muse-image** 是一个 Claude Code skill。名字来自古典神话中"在艺术家耳边低语的灵感女神"。

当你说"想画一个 xxx"，传统文生图工具会追问你"什么风格 / 什么颜色 / 什么构图"——你需要从零想象所有细节，这对大多数人是困难的。Muse-image 反过来：**它扮演一位精通绘画语言的专业艺术家**，直接替你扩写出一份 4–7 段的完整画面方案，每段 60–180 字，密度高、画面感强。然后逐段带你审阅。

每段不只让你确认，还给你 2–3 个"也可以是这样"的差异化备选方向——有时换个氛围、有时换种视角、有时换个文化语境。你没灵感时它替你想，你有想法时一句话就能精准调整。最终输出一份高质量 prompt 文件，可直接用于各类文生图平台。

## 核心思路

> 不是"AI 帮你画图"，而是"AI 把你心里那张图画清楚"。

大多数人的困境不是"不会写 prompt"，而是"不知道想要什么画面"。Muse-image 的设计哲学是：**AI 先交付一个具体、可视的方案，用户只需点头、摇头、或被备选方向启发**。你做主编，AI 做提案者。

工作流程：

```
你说一句话想法
    ↓
AI 艺术家内部一次写完完整画面方案（4-7 段）
    ↓
逐段展示给你审阅——每段附带 2 个差异化的备选方向
    ↓
你对每段：✅ 接受  /  🎨 换备选方向  /  ✏️ 自由修改
    ↓
全部确认 → 写入 .prompt.md 文件
    ↓
选择：出图（调 API）/ 只要 prompt（复制到其他平台）
```

## 与直接写 prompt 的区别

| | 直接写 prompt | Muse-image |
|---|---|---|
| 起点 | 你需要清楚知道想要什么画面 | 你只需一句脑洞，AI 替你想完整方案 |
| 灵感来源 | 全靠自己 | AI 每段提供 2–3 个备选方向启发你 |
| 描述密度 | 依赖于你的写作能力 | AI 以专业艺术家的文笔扩写，60–180 字/段 |
| 负向词 | 需要自己知悉并填写 | 三级自动叠加（默认 + 风格反义 + 动态推导） |
| 风格选择 | 需要记忆或查询风格名 | 展示 4 大类 50 种风格，可混搭 |
| 迭代 | 自己反复修改文本 | 对话式逐段确认，精准定位要改的段落 |

## 快速上手教程

### 第一步：安装

```bash
# 将仓库复制到 Claude Code 技能目录
cp -r Muse-image ~/.claude/skills/muse-image/

# （可选）配置 API 以支持直接出图
cd ~/.claude/skills/muse-image/
python3 setup.py
```

如果不配置 API，依然可以使用全部 prompt 生成功能，最后将 prompt 复制到 Midjourney / Stable Diffusion / 即梦 / 文心一格等平台出图。

### 第二步：启动

在 Claude Code 对话中直接输入：

```
/muse-image
```

然后告诉它你想画什么。例如：

- "一只灰蓝色短毛兔，戴圆框眼镜，在咖啡馆敲 MacBook"
- "新中式水墨风的侠客，站在竹林中，阴森但克制"
- "现代艺术风格的缪斯女神，体现灵感与艺术"
- "赛博朋克城市的夜景，雨刚停，霓虹灯映在水洼里"

### 第三步：回答一个可选问题

AI 会问你是否有特别的风格、比例或禁忌。如果你已经有了想法（比如"吉卜力风格"、"16:9 横版"），直接说；如果没有，回车跳过——AI 会根据自己的判断选最合适的方案。

### 第四步：逐段审阅画面方案

AI 会逐段展示构思。每段格式如下：

```
【1. 主体细节】
一段 60–180 字的具体画面描述，包含可视化的名词与动词……

备选方向：
A. "清冷晨光独处感" —— 角色改为清晨空荡空间里唯一的身影
B. "夜雨窗内孤光感" —— 窗外下雨，室内仅一盏灯照亮角色
```

你的选择：
- **✅ 就这样，继续** — 接受，看下一段
- **🎨 换方向 A / B** — AI 按该方向重写本段，然后重新过审
- **✏️ 我改一下** — 输入修改意见（如"把咖啡杯换成抹茶碗"）

如果选备选方向，AI 会完整重写整段并再生成新的备选。

### 第五步：选择输出方式

全部段落确认后，AI 将完整 prompt 写入文件并让你选择：

| 选项 | 效果 |
|---|---|
| **✅ 出图** | 调用 gpt-image API 生成图像，输出 PNG 文件 |
| **📝 只要 prompt** | 保存 prompt 文件，不调用 API |
| **✏️ 改一下** | 自由描述要改的内容，修改 prompt 后重新确认 |

选「只要 prompt」时，得到的 `.prompt.md` 文件可以：
- 直接复制正文到 Midjourney / Stable Diffusion / ComfyUI 使用
- 用任意文本编辑器修改后再用
- 收藏归档，作为灵感素材库

## 进阶用法

### 命令行快捷模式

如果你已经清楚想要什么，可以跳过对话流程直接出图：

```
/muse-image --style 19 a cyberpunk alley at night
/muse-image --style 25+18 --size 16:9 一只猫在蒸汽朋克城市探险
```

### 复合风格混搭

风格 ID 可以用 `+` 组合（主风格 + 辅风格）：

```
/muse-image --style 25+18 ...   # 吉卜力 × 蒸汽朋克
/muse-image --style 44+27 ...   # 中国水墨 × 印象派
```

### 使用参考图

在初始描述中附上本地图片路径，AI 会读取图像并提取视觉特征（色调、构图、质感）拼入 prompt：

```
/muse-image
> 参考这张图的风格，画一只坐在窗边的黑猫
> 参考图：D:\photos\reference.jpg
```

注意：gpt-image-2 API 不支持图像输入，参考图仅用于提取文字描述。

### 修改已生成的 prompt

生成的 `.prompt.md` 文件可用任意文本编辑器直接修改。改完后两种方式出图：

**方式一**：重新回到 Muse-image 对话，选「✏️ 改一下」，AI 读取修改后的文件继续。

**方式二**：直接命令行调 API：

```bash
python3 scripts/gen_assets.py \
  --prompt-file prompts/latest.prompt.md \
  --size 9:16 \
  --out my_image.png
```

### 在第三方平台使用 prompt

`.prompt.md` 文件结构：

```markdown
---
run_id: ...
style_id: ...
(YAML 元数据，AI 生成平台忽略)
---

主体细节:
(中文分节描述，可直接复制正文)

画面风格(style anchor):
Flat design illustration style, Bauhaus geometric composition.  (英文风格锚定)

Avoid: ugly AI-generated look, ...  (英文负面抑制词)
```

**不同平台的使用方式**：

| 平台 | 操作 |
|---|---|
| **Midjourney** | 复制正文 + 风格锚定 + Avoid 段，拼接为英文 prompt。将 Avoid 内容改写为 `--no` 参数 |
| **Stable Diffusion** | 英文 prompt 放正向框，Avoid 段内容放负向框 |
| **ComfyUI** | 同上，分别接入 CLIP Text Encode (positive) 和 (negative) 节点 |
| **即梦 / 文心一格** | 复制中文分节正文即可，平台会自动理解画面描述 |
| **DALL·E / ChatGPT** | 直接将英文 prompt 整体粘贴发送 |

## 三级抑制体系

Muse-image 自动叠加三层负面提示词，防止 AI 出图产生常见缺陷：

### L1 — 默认全局抑制
所有 prompt 自动附加：塑料皮肤、变形手指、水印、签名、JPEG 伪影、低分辨率、模糊、库存照片感……

### L2 — 风格反义抑制
根据所选风格自动选择对立的负面词：

| 风格群 | 自动抑制 |
|---|---|
| 写实摄影 | 卡通脸、Low-poly 几何、无深度的平面着色 |
| 像素/8位 | 照片噪点、平滑抗锯齿、柔和虚焦 |
| 动漫/吉卜力 | 写实皮肤毛孔、硬影棚阴影、粗糙写实感 |
| 极简/扁平 | 密集细节、重阴影、写实质感 |
| 油画/水墨 | 喷枪平滑感、矢量边界、霓虹光效 |
| 赛博/合成波 | 暗淡色调、大地色系、手绘瑕疵 |
| 3D 渲染 | 笔触纹理、手绘线条、纸张质感 |

### L3 — AI 动态推导
每次生成时 AI 基于你的内容 × 风格 × 氛围的实际组合，额外生成 2–4 条针对性抑制。例如"活泼亲切"主题会自动避开"阴森氛围、硬对比度、企业严肃感"。

### L4 — 用户自定义
你在对话中提到的任何禁忌（"不要太萌"、"不要现代建筑"）都会被英文化后拼入 Avoid 段。

## Prompt 文件格式说明

每次合成的 `.prompt.md` 文件结构如下：

```yaml
---
run_id: 20260521_003526          # 唯一标识（时间戳）
created_at: 2026-05-21T00:35:26  # 生成时间
style_id: "49"                   # 风格 ID（可复合，如 "25+18"）
aspect_ratio: 9:16               # 画面比例
ai_dynamic_suppression: [...]    # AI 动态生成的 L3 抑制词
---
```

正文部分按视觉逻辑从主体 → 行为 → 环境 → 表面属性 → 排版排列，每段以中文小标题引导。末尾附英文风格锚定句和 Avoid 负面抑制段。

**编辑提示**：只修改 YAML 头以下的内容。`run_id`、`created_at` 等元数据仅供追溯，不影响出图。

## 常见问题

**Q: 没有 API key 能用吗？**
完全可以。核心价值（艺术家扩写 + 分段审阅 + 抑制体系）和图像 API 是解耦的。选"📝 只要 prompt"即可拿到完整的 prompt 文件。

**Q: prompt 可以在其他平台用吗？**
可以。正文是中文分节描述，风格锚定和 Avoid 段是英文。Midjourney 用户把内容拼接为英文 prompt，将 Avoid 改为 `--no` 参数；SD 用户分别放入正向和负向框。

**Q: 怎么让生成的画面更接近我想要的？**
在逐段审阅时，不要只选"就这样"——积极尝试备选方向。没灵感时备选方向是最好的启发器；有明确想法时直接选"✏️ 我改一下"精准指出要改什么。

**Q: 一次会话能生成几个版本？**
没有硬性上限。对不满意的段落可以反复换备选方向或手动修改，最终确认后也可以选"✏️ 改一下"整体调整。每次调整覆盖同一个 prompt 文件。

**Q: 之前生成的 prompt 会丢吗？**
不会。每次生成使用时间戳命名（如 `20260521_003526.prompt.md`），新生成不影响旧文件。`latest.prompt.md` 是指向最新版本的快捷入口。

**Q: API 图像生成失败怎么办？**
检查 `config.env` 中的 API key 和 base URL 是否正确。如果 API 不可用，不影响 prompt 生成——选"📝 只要 prompt"，拿 prompt 文件去其他平台出图即可。
